
Программа распознавания лиц статических изображений и видео. В качестве движка распознавание лиц используется библиотека dlib (http://dlib.net).

Требования:
dlib: 19.17
OpenCV: 4.1.0

Проекты MSVS 2015 сборки для Windows:

face.vcxproj - проект без использования CUDA
face.cuda9.vcxproj - проект для CUDA 9.0
face.cuda10.vcxproj - проект для CUDA 10.0

Файлы dlib должны находится в папке C:\dlib, но собирать нет необходимости, т.к. собранные файлы dlib.lib включены в этот проект.

Если собирается проект с использованием CUDA и CUDNN, в окружении должна быть задана переменная CUDA_PATH

В окружении должна быть задана переменная OPENCV_DIR

Makefile - сборка для Linux, пока не доведён до конца (нерабочий).

Файлы нейросетевых моделей детекции и распознавания лиц находятся в папке build.

Уже собранные исполняемые файлы для Windows положены в папку build.

При запуске файлы нейросетевых моделей должны находится в одной папке с исполняемым файлом.


Использование.
(в папке build/samples есть рабочие примеры)


Параметры командной строки:

Общие:

-logfile
указывается путь и имя файла для записи лога, по умолчанию - не сохраняется, выводится только лог на экран (в stdout)

-framepath
указывается путь для сохранения кадров с лицами, по умолчанию - текущая папка

-personpath
указывается путь к файлу таблицы персон (persons.csv), по умолчанию - текущая папка

-update
ничего не указывается, разрешает обновление таблицы персон

(указываемые пути должны существовать, автоматически они не создаются)


Режим статических изображений:

-image
указывается список файлов изображений

-person
указывается имя персоны при этом разрешается обновление таблицы персон

-out
(только при выключенном режиме обновления таблицы персон) указываются имена файлов с распознанными лицам, порядок соответствует порядку файлов в -image


Режим видео:

-video
указывается видеофайл иди видеопоток 

-frameskip
указывается количество пропускаемых кадров для распознаний, по умолчанию – 5. Увеличение пропускаемых кадров - повышает скорость работы, но и повышает требования ко времени непрерывного нахождения номера на видео. Быстрое движение объекта требует снижения числа пропускаемых кадров, но возрастают аппаратные требования

-buffersize
указывается размер буфера для накопления кадров, по умолчанию – 50. Увеличение размера буфера увеличивает скорость работы при использовании CUDA, но увеличивает расход памяти

-showon
ничего не указывается, включение окна видео, по умолчанию - не включено
 
-kalmanoff
ничего не указывается, отключение фильтра Калмана для позиций рамок номеров на выходном видео, по умолчанию - включено 


Формат именования сохраненяемых кадров с лицами:
<person id>-<face track id>-<x>-<y>-<w>-<h>.jpg (нарисована рамка для лица)

Формат записи лога:
<person id>;<face track id>;<x>;<y>;<w>;<h>;<start frame>[<start timecode>]<end frame>[<end timecode>];<facedescriptor>;<person name>

Формат строки в таблице персон:
<person id>;<count>;<standart deviation>;<facedescriptor mean>;<person name>
(имена персон можно редактировать в файле таблицы персон, но в этой программе они нигде не отображаются, только присвоенные идентификаторы)


Примеры:

Вносится в таблицу персона (во всех изображениях должны быть не менее 10 лиц одной и той же персоны)
face.exe -image pic1.jpg pic2.jpg pic3.jpg -person "Person Name"

Распознаются несколько изображений:
face.exe -image pic1.jpg pic2.jpg pic3.jpg -out res1.jpg res2.jpg res3.jpg

Распознаётся видеопоток или видеофайл, новые персоны заносятся в таблицу персон:
face.exe -video https://domain/videostream -update
face.exe -video rtmp://videostream -logfile ./log.txt -framepath ./persons -personpath ./persons -showon -kalmanoff -frameskip 3 -buffersize 20 -update
face.exe -video sample.avi -logfile ./log.txt -framepath ./persons -personpath ./persons -showon -kalmanoff -update
